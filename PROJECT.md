# AI Media Service — Описание проекта

## Что это

Автономный AI-агент для ведения трёх Telegram-каналов на русском языке.
Система сама генерирует контент, отправляет тебе на проверку и публикует после одобрения.
Никакого ручного написания постов — ты только нажимаешь ✅ или ❌.

---

## Каналы

| Ниша | Канал | О чём |
|------|-------|-------|
| `ai_business` | Нейро Бизнес | AI-инструменты, автоматизация, кейсы, промпты |
| `personal_brand` | Виден всем | Рост аудитории, упаковка, монетизация |
| `finance` | Деньги просто | Бюджет, накопления, инвестиции |

---

## Как работает

```
Каждый будний день 08:30
        │
        ▼
Выбирается ниша по ротации (пн/чт=AI, вт/пт=Личный бренд, ср/сб=Финансы)
        │
        ▼
ChiefAgent запускает цепочку субагентов:
  1. NicheAnalystAgent    → анализирует тренды и боли аудитории
  2. ContentStrategyAgent → составляет план из 4 тем на неделю
  3. ContentGeneratorAgent→ пишет статью + 3 Telegram-поста + 3 сценария Shorts
  4. QABrandSafetyAgent   → проверяет контент на качество и безопасность
        │
        ▼
Контент сохраняется в БД
Тебе приходит уведомление в Telegram с превью и кнопками ✅ Approve / ❌ Reject
        │
        ▼
Ты нажимаешь Approve → пост попадает в очередь
        │
        ▼
В 10:00, 15:00 и 19:00 система публикует следующий пост из очереди в канал
```

## Агенты — кто за что отвечает

### ChiefAgent
Главный оркестратор. Получает задачу, планирует порядок действий, вызывает субагентов через function calling и собирает итог.
При rate limit мгновенно переключается: **Groq → Gemini → Mistral**.

### NicheAnalystAgent
Аналитик рынка. Находит топ-3 тренда прямо сейчас, выявляет боли аудитории, ищет темы с высоким потенциалом.

### ContentStrategyAgent
Стратег. Превращает аналитику в конкретный план: 4 темы на неделю с форматом, эмоцией и хуком для каждой.

### ContentGeneratorAgent
Автор. По одной теме сразу пишет три формата:
- **Статья** 700–1000 слов (SEO, с заголовками H2, CTA в конце)
- **3 Telegram-поста** (инсайт / кейс / чеклист, макс 200 слов каждый)
- **3 сценария Shorts** (15–45 сек: хайп → суть → призыв)

### QABrandSafetyAgent
Редактор. Проверяет: нет ли непроверенных фактов, нет ли запрещённого контента в РФ, есть ли CTA, понятен ли текст. Возвращает pass/fail с конкретными замечаниями.

### GrowthMetricsAgent
Аналитик роста. Анализирует метрики (охват, ER, CTR) и решает что оставить, что убрать, предлагает 2 гипотезы для A/B теста.

### SeoAgent
SEO-специалист. Подбирает ключевые запросы для Яндекса/Google, пишет мета-теги, структуру H1–H3.

### TelegramEditorAgent
Редактор канала. Адаптирует любую идею под Telegram: хук в первой строке, живой язык, умеренные эмодзи, CTA.

### ShortsScriptAgent
Сценарист. Пишет сценарии для Reels/Shorts/TikTok: структура ХАЙП → СУТЬ → ПРИЗЫВ, тон дерзкий и прямой.

# Что дорабатывать и как развивать проект

## Шаг 3 — Telegram-бот для управления (следующий приоритет)

Бот запущен, но команды минимальны. Нужно добавить:

- `/status` — сколько постов в очереди, статус провайдеров, когда следующая генерация
- `/queue` — список всех pending постов с возможностью апрува прямо из бота
- `/skip` — пропустить текущий пост в очереди без публикации
- `/generate ai_business` — запустить генерацию для конкретной ниши прямо сейчас
- `/report` — сводка за неделю: сколько постов опубликовано, по каким нишам

---

## Шаг 4 — Метрики и обратная связь

Сейчас GrowthMetricsAgent работает вхолостую — нет реальных данных.

**Что сделать:**
- Подключить Telegram Bot API для получения статистики канала (просмотры, подписчики)
- Сохранять метрики в `WeeklyMetricsDB` автоматически раз в неделю
- GrowthMetricsAgent получает реальные цифры → даёт реальные рекомендации
- Еженедельный отчёт в Telegram: что сработало, что нет, план на следующую неделю

---

## Шаг 5 — Улучшение качества контента

Текущий контент генерируется хорошо, но можно улучшить:

- **Добавить актуальность**: перед генерацией делать поиск новостей по нише (Serper API или Tavily) — NicheAnalystAgent будет ссылаться на реальные события
- **Изображения**: генерировать обложки для постов через DALL-E или Stable Diffusion
- **Длинные статьи**: публиковать на Telegraph и вставлять ссылку в Telegram-пост
- **Опросы**: TelegramEditorAgent уже придумывает вопросы — добавить автопубликацию опроса вместе с постом

---

## Шаг 6 — Расширение ниш

Сейчас 3 ниши. В `niches.py` уже описаны ещё 7:

```
career, habits, education, relationships, travel, home, tech_news
```

Чтобы добавить нишу:
1. Создать Telegram-канал
2. Добавить `TELEGRAM_CHANNEL_XXX` в `.env`
3. Добавить в `settings.niche_channels` в `config.py`
4. Добавить в `choose_pilot_niches()` в `niches.py`
5. Добавить в ротацию в `worker.py` (`_NICHE_BY_WEEKDAY`)

---

## Шаг 7 — Монетизация и интеграции

- **Партнёрские ссылки**: добавить в систему-промпт список партнёрок для каждой ниши — агент будет органично вставлять в CTA
- **Лид-магниты**: генерировать мини-гайды (PDF) по топовым темам, собирать базу через бота
- **Рекламные вставки**: отдельный шаблон для рекламных постов, которые ChiefAgent вставляет раз в 5–7 постов
- **VK / Дзен**: добавить адаптеры для кросс-постинга (сейчас только Telegram)

---

## Шаг 8 — Масштабирование

- **OpenRouter** как 4-й резервный провайдер (ключ уже в `.env`, просто добавить в цепочку)
- **Несколько воркеров**: запустить 3 Celery-воркера параллельно — по одному на нишу
- **Приоритеты очереди**: срочные посты (новости) идут вне расписания
- **A/B тесты**: GrowthMetricsAgent предлагает гипотезы → система автоматически тестирует два варианта заголовка и выбирает лучший по охвату
